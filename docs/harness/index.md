# Harness Index

Harness engineering means making the repository easy for agents to inspect, run, validate, and repair without relying on hidden human context.

## Local Harness

- Local development: `local-dev.md`
- Browser validation: `browser-validation.md`
- Runtime observability: `observability.md`
- CI and repo-health gates: `ci.md`

## Harness Principles

- Keep `AGENTS.md` short and index-driven.
- Put durable product and engineering knowledge in repository-local docs.
- Prefer executable checks over prose rules.
- Make browser behavior, logs, and verification artifacts legible to agents.
- Promote repeated review feedback into scripts, tests, lint rules, or indexed docs.
