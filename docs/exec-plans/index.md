# Execution Plans

Execution plans are first-class artifacts for durable multi-step work. They replace the old sprint-row protocol.

## Directories

- Active plans: `active/`
- Completed plans: `completed/`
- Templates: `templates/`

## When To Create A Plan

Create a checked-in plan when work is multi-step, cross-layer, risky, audit-like, migration-like, or likely to span multiple agent turns.

For small isolated fixes, an in-chat plan plus final verification evidence is enough.
